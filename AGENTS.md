# Instrucoes locais do DuckHat

## Leitura obrigatoria para revisao funcional

Antes de revisar funcionalidades, mapear comportamento de telas, entender fluxo de botoes ou decidir impacto de mudancas em arquivos Flutter e backend, ler primeiro:

`docs/funcionalidades-por-arquivo.md`

## Objetivo desse arquivo

Esse documento concentra:

- funcao de cada arquivo principal do projeto
- telas e subpaginas existentes
- botoes, CTAs e navegacao entre paginas
- o que esta integrado com API
- o que ainda esta mockado ou placeholder

## Regra de trabalho

- Nao assumir funcionalidade pela memoria sem consultar `docs/funcionalidades-por-arquivo.md` e o codigo real.
- Se uma funcionalidade mudar, atualizar esse documento no mesmo trabalho sempre que a mudanca afetar comportamento, fluxo ou responsabilidade de arquivo.
- Para revisoes funcionais, usar esse documento como indice inicial e depois abrir os arquivos reais impactados.
